01 · Setup

Stack y dependencias.

El sistema es framework-agnóstico. Funciona con HTML estático, Next.js, Astro, Remix o cualquier setup que permita CSS-in-CSS. Cero runtime, cero JS obligatorio (excepto el theme toggle).

Fonts (Google Fonts)
  • Fraunces Display · variable · opsz 9-144 · wght 400-700 · SOFT 0-100
  • Manrope Body / UI · variable · wght 300-700
  • JetBrains Mono Code / numbers · wght 300-700
Browser support
  • Safari 15+ (color-mix, oklab)
  • Chrome / Edge 111+
  • Firefox 113+
  • iOS Safari 15.4+
  • Android Chrome latest

Instalación

CSS base · copia y pega.

Pega este bloque en tu app.css o global.css. Define la totalidad de tokens, family fonts y semantic mapping con soporte light/dark.

/* Dark via system preference */
@media (prefers-color-scheme: dark) {
  :root:not([data-theme="light"]) {
    --bg: var(--n-900);
    --bg-elev: var(--n-800);
    --surface: var(--n-700);
    --fg: var(--n-50);
    --fg-soft: var(--n-200);
    --fg-muted: var(--n-300);
    --brand: var(--g-300);
    --brand-soft: var(--g-400);
    --accent: var(--b-300);
    --line: var(--n-700);
    --line-strong: var(--n-600);
  }
}

/* Manual override */
[data-theme="dark"] { /* same dark tokens */ }
Por qué :root:not([data-theme="light"]) dentro del media query: permite que el sistema decida por default pero que el usuario fuerce light mode con data-theme="light" en <html>. Lo mismo para forzar dark.

Accesibilidad

Checklist WCAG 2.2 AA.

  • Contraste de texto. Body 14.6:1, accent en cream 5.8:1. Verificar en dark mode.
  • Focus visible. Todos los elementos interactivos con outline: 2px solid var(--accent); outline-offset: 3px.
  • Reduced motion. Envolver animaciones en @media (prefers-reduced-motion: no-preference).
  • Skip link. Primer elemento del body es link "Saltar al contenido principal".
  • Heading hierarchy. Una sola h1 por página. No saltar niveles.
  • Form labels. Todo input asociado a su label vía for / id o wrapping.
  • Alt text. Imágenes informativas con alt descriptivo. Decorativas con alt="".
  • Language. <html lang="es-MX"> obligatorio.
  • Color scheme. <meta name="color-scheme" content="light dark"> para que el browser respete el modo.
/* Respeto a usuarios que prefieren menos movimiento */
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
Performance

Presupuesto y prácticas.

  • CSS único. Mantener bajo 30 KB minificado (este sistema completo < 18 KB).
  • Fuentes. display=swap obligatorio. Preconnect a Google Fonts.
  • Imágenes. WebP / AVIF con fallback. Lazy-load en imágenes below-the-fold con loading="lazy".
  • Logo. SVG inline para el logo de header (cero round-trip). PNG para emblema en cards.
  • JS. Solo theme toggle y IntersectionObserver para reveals. < 2 KB total. No frameworks en landing.
  • Caché. Cache-Control: public, max-age=31536000, immutable para assets versionados.
  • CLS. Reservar espacio para imágenes con aspect-ratio. Cero layout shift acumulado.
  • Critical CSS. Inline above-the-fold (~6 KB). El resto async.
